
# üîê **Data Security and Governance ‚Äì Theory**

Data Security & Governance ensures that **machine learning initiatives protect sensitive data, comply with legal regulations, and minimize risk to the organization.**
This is crucial especially in industries like **finance, healthcare, and insurance**, where customer data is extremely sensitive.

### **1Ô∏è‚É£ Data Privacy and Data Security**

**Data Privacy** ‚Üí focuses on **how personal data is collected, processed, stored, and shared** according to laws and user consent.

**Data Security** ‚Üí focuses on **protecting data from unauthorized access, attacks, and data breaches**.

Important principles:

| Privacy               | Security                    |
| --------------------- | --------------------------- |
| Why data is collected | Encryption                  |
| Who has access        | Role-based access           |
| Consent               | Secure API communication    |
| Data minimization     | Network firewalls, VPC, IAM |

Real techniques used in MLOps:

* Encryption **in transit (TLS/HTTPS)** and **at rest (AES-256)**
* Masking / Tokenization for PII fields (name, phone number, bank details)
* Access logging & audit trails
* Zero-trust security models

### **2Ô∏è‚É£ Data Access Management**

Not every engineer should access every dataset.

Control using:

| Method                 | Purpose                                 |
| ---------------------- | --------------------------------------- |
| IAM Roles              | Restrict access based on responsibility |
| RBAC                   | Project-wise or team-wise data access   |
| Attribute-based access | Time/location/department-based access   |
| Secrets Management     | Store credentials securely              |

Tools:

* AWS IAM / GCP IAM / Azure AD
* Vault, AWS Secrets Manager, GCP Secret Manager

Golden rule:
‚ö†Ô∏è **Never store access keys inside code, GitHub, or Docker images**

### **3Ô∏è‚É£ Data Retention**

Data Retention defines **how long data should be stored and when it should be deleted**.

Why retention matters:

* To comply with laws & customer policies
* Reduce storage cost
* Prevent misuse of old data

Typical retention approaches:

* Raw data ‚Üí stored short duration
* Feature engineered data ‚Üí stored longer
* Aggregated and anonymized data ‚Üí stored longest

Retention policy depends on:

| Industry   | Typical Duration |
| ---------- | ---------------- |
| Healthcare | 6‚Äì10 years       |
| Banking    | 7 years          |
| E-commerce | 1‚Äì5 years        |

### **4Ô∏è‚É£ Need of Compliance & GDPR**

Organizations must follow compliance laws to **protect customer identity and avoid legal penalties**.

**GDPR ‚Äî General Data Protection Regulation (EU)**
Rules:

* Collect minimum data needed
* Inform users how data will be used
* Allow users to delete/export their data (`Right to be Forgotten`)
* Report any data breach within **72 hours**

Impacts on MLOps:

* Personal data should be anonymized before ML training
* ML monitoring systems cannot expose identity-linked logs

---

### **5Ô∏è‚É£ HIPAA Compliance**

Applies to **Healthcare data in the United States**.

Regulates:

* Medical history
* Diagnosis & prescriptions
* Lab reports
* Insurance health data

Key requirements for ML workflows:

* Data encryption at every stage
* Secure patient identity masking before ML training
* Strict access controls for sensitive datasets
* Audit logs for every data access

---

### **6Ô∏è‚É£ PCI Compliance**

Applies to **payment card & banking data**.

Covers:

* Credit/Debit card numbers
* CVV
* Bank account details
* Transaction logs

PCI DSS rules:

* No storing raw CVV ever
* PAN (card number) must be encrypted/masked
* Redaction required in logs and datasets
* Payment systems must undergo periodic security testing

---

### **7Ô∏è‚É£ Compliance Consequences & Penalties**

If companies **ignore compliance**, consequences can be extremely costly.

| Violation Type | Risk                                        |
| -------------- | ------------------------------------------- |
| GDPR           | Millions of euros in fines                  |
| HIPAA          | Criminal charges + business shutdown        |
| PCI DSS        | Heavy fines + blocking payment facilitation |
| Data breach    | Loss of customer trust + lawsuits           |

Example headlines in real world:

* Hospitals fined for exposing patient reports
* Banks penalized for storing unencrypted transaction logs
* E-commerce companies sued for selling user analytics without consent
